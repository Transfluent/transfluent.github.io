When placing your order, you may provide callback_url parameter. A HTTP POST request with JSON payload will be made to the callback URL once translation is completed.

Example callback for text orders:

{% highlight json %} { "text_id" : "example-translation-key", "group_id" : "Test-App", "text" : "Vive la France !", "language" : { "name" : "French (France)", "code" : "fr-fr", "id" : 2, "dir" : "ltr" }, "source_language" : { "name" : "English (Great Britain)", "code" : "en-gb", "id" : 1, "dir" : "ltr" }, "level" : {"code" : "economy", "int" : 8}, "previous_text" : "Vive la France!" } {% endhighlight %}

Example callback for file orders:

{% highlight json %} { "updated_files": { "4": "https:\/\/transfluent.com\/v2\/file\/read\/?token=TOKEN-GOES-HERE&identifier=my-project-Localizable.strings&language=zh-cn" }, "languages": { "4": "zh-cn" } } {% endhighlight %}

Security

It is highly recommended to include a secret key to the callback URL. E.g. https://example.org/my-app/transfluent-callback?auth-key=ABC123 where auth-key parameter contains your secret that must match before your application starts processing any callback. This provides security against third party providing unauthorized content to your system.
