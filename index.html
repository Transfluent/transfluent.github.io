Transfluent is the only translation provider..

..to scale from 1 word into millions of words, yet providing excellent quality and having ability to translate into numerous of languages with very little effort from the customer.

Key to fulfill this promise is our innovative use of technology in the translation process. Our API is in the core enabling all this. On this website you will find detailed documentation of our API and description how to the translation process works.

Get started

All of our implementations are built on our API. The translation process over our API is straight forward and simple: authenticate, submit content, place the order and receive the translation (via an API call or by receiving a callback).

User accounts

To get going, you will need a Transfluent user account. Either create a new user account by yourself or ask your company's account owner to create one for you.

Development and testing

While developing and testing your integration, you probably do not want to spend your own money for testing. Just for that, we provide a Pseudo language translation. The language code for pseudo language is xx-xx and the language id is 500. Use English (en-gb) as source language and Pseudo language as target language and the order will be processed for free. Instead of actual translation, you will get your source text reversed (e.g. "Hello" -> "olleH"). Placing Pseudo language orders makes sense for testing the whole process from end to end and depending on your use case possibly to pinpoint any non-translated parts on your end.

Supported languages

Check supported language pairs from our website or call our API's languages method to list languages names and language pairs. The API methods can be called using ready-made clients or command line tools, for example call using curl: curl https://transfluent.com/languages/ or you can even do it with your browser: HTTP GET https://transfluent.com/languages/. The output is always JSON. As you can see, it is easy to get started with the API.

Use cases

  • API v2/text*: Plain text and HTML content. Content is stored by key (and group). Our backend keeps track of versions, so unchanged texts will not be re-translated.
  • API v2/file*: Resource files. Usually integrated to the application build process. Our backend keeps track of file versions, so unchanged parts will not be re-translated.
  • Translation Widget: ad-hoc translation needs, delivery by email. Text or files. Embedded version available for intranet.

Please note

We would like to remind you of few important points that might cause issues later on if not addressed early on.

  • Charset encoding. We always provide UTF-8 or UTF-16 encoded content. We highly recommend to use UTF-8.
  • Writing direction. Some languages are written from right to left, e.g. Arabic.
  • Plurals. Some languages have more plural forms than two.
  • Space. Some languages need more space in length on a screen and also in bytes on a storage device. This is utmost important if your translation has either limited storage length or if the text must fit on screen. We provide a tool to estimate translation size for AppStore descriptions, it can be used for other estimation purposes as well.
  • Tags/placeholders. Instead of combining strings (e.g. "Hello " + name), prefer using tags and placeholders (e.g. " Hello [name]"). That is because some language have different order of words and therefore combining strings might be problematic. Our translation system supports and our translators are familiar with the most common tags for each resource type but if you are translating plain text, it might be worthwhile to mention about your tags/placeholders in the translator instructions.
  • Tone&context. Please provide as accurate instructions and supporting context as you can, it will help the translators to do a better job.
  • Hardcoded content. To fully localize a website or a game, you might encounter issues with some content being "hardcoded" in the source language. This is usually the case with images and strings that are not passed through the localization system. To identify these, you can use our Pseudo language as target language and then see if all content gets reversed or not.
